git指令

git status [-s]：查看本地文件状态
		   [-s]: 简介方式查看
git add <文件名/.>：将文件添加到暂存区(.表示左右文件)
git commit -m “注释”：将暂存区文件添加到版本库

git reset --hard HEAD^：工作区文件回退到上一次提交的版本，git log记录也会删除
				 HEAD^^:工作区文件回退到上上一次提交的版本
				 HEAD~100：工作区文件回退到前100个版本
				 版本号：工作区文件回退到版本号的版本
				 
git reflog：管理历史提交版本号		

- 在未add前为返回到未修改暂存区的状态
- 在add后为返回的状态
- 在未commit之前撤销删除操作
git restore [文件名]
git checkout -- [文件名] 

- 将本地仓库可远程仓库进行绑定
git remote add origin github.com/mhvhmj/learn-test

- 取消与远程仓库的绑定
git remote remove origin

- 将本地的 master 分支推送到 origin 主机的 master 分支
git push origin master

- 创建分支
git branch <分支名>
不同的分支间的文件相互独立

- 删除分支
git branch -d <分支名>

- 切换到指定分支-切换分支后文件夹内容也随之改变
git checkout <分支名>
- 创建分支并切换到新建分支
git checkout -b <分支名>
- 创建远程仓库的分支到本地，并切换到那个分支
git checkout -b <分支名> origin/<与前面的分支名一致>

- 查看当前分支
git branch

- 将<分支名>下所有提交的文件合并到当前
	- 同名文件内容冲突的会将显示每个分支的内容，需要进行add-commit操作，不然当前分支会显示(master|MERGING)
	- 不存在的文件则会进行复制
	- <分支名>文件不会与当前分支文件合并
git merge <分支名>

- 查看某一分支下的文件
git show <分支名>:<文件路径>


- master分支应该是最稳定的版本一般情况下不允许在上面编辑，代码编
  辑一般情况下在新建的dev分支上干活，干完后，比如上要发布，或者
  说dev分支代码稳定后可以合并到主分支master上来。
  
- 隐藏了工作现场
git stash

- 显示隐藏的工作现场
git stash list

- 删除隐藏的工作现场
git stash drop

- 查看远程库的信息
git remote [-v]
			-v:查看远程库的详细信息

- 将分支上的所有本地提交到远程库中,并将远程仓库与本地合并
git push <远程仓库> <本地分支>

- 分支推送原则：
	- master分支是主分支，因此要时刻与远程同步。
	- 一些修复bug分支不需要推送到远程去，可以先合并到主
		分支上，然后把主分支master推送到远程去。
